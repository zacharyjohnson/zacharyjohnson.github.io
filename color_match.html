Color Match 🎨

Interaction Design, Visual Design, iOS Development

Introduction

In the summer of 2017, after completing my freshman year at Purdue, I wanted to dive into a project to practice my user interaction, design, and development skills with a simple iOS game.

Inspiration

I’m fascinated by color theory. The complexities, interactions, and components of color are all incredibly interesting to me, so I wanted to create a game that used color as the main component.

Target Users

I created a target user group to help set my goals for the game. Admittedly, this is based purely on my own assumptions and goals for the project, as I did not interview or research potential users.

Target Users: Casual mobile gamers
User needs:

  • Quick, easy, accessible gameplay
  • Short burst of entertainment
  • Goal-oriented gameplay

Differentiator: Not necessarily concerned with story, plot, life-like graphics, or character development

Game Design

My initial focus was on the gameplay. After some brainstorming on how to incorporate colors into a game, I decided to present the user with a gradient of random colors and the task to find a certain color in a progressively shortening period of time. I started sketching how the game would be set up:

I wanted to use the interaction on the screen as a component to increase difficulty. In the first, easiest mode of the game, a white bar containing a color and empty slot greets the user. This limits the difficulty by creating one-dimensional movement along the x-axis. In more challenging modes, two-dimensional gradients and a circle cause the player to find the correct color in a larger, more complex gradient through moving the circle along both the x and y axis.

User Journey

With the goal of maintaining simplicity and ease to use for casual games, I tried to keep the number of screens minimal. I created a simple flow between a start screen and the game, with the start screen containing a scroll wheel to select the game mode. Though slightly unconventional for a game, I decided that the scroll wheel was the easiest method for a user to select a game mode and start the game from one screen.

Wireframing

Next, I created some wireframes to refine the individual screens. While creating the wire frames, I refined the design to maintain usability, but also increase the overall visual appeal of the game. For example, though I believe a scroll wheel allowed for ease of use, two buttons for the various game modes is more well-known by users while also using the space on the screen more appropriately.

Mockups

I created a very sparse a set of eight colors that could be randomly selected to create a gradient for the game and a font to use for headers and text.

Outcome

As this project was started as more of a coding and development project, I began developing a working prototype in XCODE with Swift. At this time, I had not fully discovered my passion for User Experience/Product design, so I did not complete many stages of the product development cycle that I wish I would have, such as initial user feedback to confirm the game, presenting the prototype friends, families, and strangers to gain feedback and iterate on the design, and narrowing the niche of the target users, but I leave this project in my portfolio as it was the spark for my passion for UI/UX design.
