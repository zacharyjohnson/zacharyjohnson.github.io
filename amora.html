Introduction

This semester, I am enrolled in CS 307: Software Engineering. This class focuses on the software development process and Scrum development. I am working with a group of 5 other Computer Science majors to develop a redesigned to-do application for desktop and mobile called Amora. I am using this webpage to journal and document all of the work that I complete on the project throughout the semester.

My Role

I am focusing entirely on the product's features, user interface, and user experience. I am the only member on the team with user experience and interface design experience, so I am bringing the human-centered design process to the project to ensure our final project has a clean, intuitive, beneficial user interface.

Amora

Our project is called Amora. It is a team task management application that allows users to join multiple teams, assign tasks to teams, assign tasks to certain time periods throughout the day, view what their teammates are currently working on, and more.

The Challenge

The challenge for this project is the time constraint. The class focuses on developing the final software application, not the product development process. I am working hard to ensure we follow the process to the best of our ability so that Amora can turn out great, but this does mean that we did not interview potential users to learn their wants; wireframe the entire application before UI mockups; nor make feature decisions based on user experience, but more on the technical complexity to develop.

Current Progress

As our team nears the start of development sprint one, we are working on a design document outlining all aspects of the application. My role for this document is to develop the user interactions, user flow, user interface, and all other aspects of the functionality of the application.

I am currently developing wireframes and high-fidelity mockups of the application. Included at the top of the page are proof-of-concept sketches of the user interface that I previously created for pitching the initial idea to project coordinators.

Stay Tuned!

As I stated, this is a continually growing project and I am using this webpage as a journal to document the process. As I finish wireframes and pixel-perfect prototypes, I will update this page.
